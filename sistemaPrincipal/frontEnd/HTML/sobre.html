SOBRE

O PROJETO

O projeto trata-se de um protótipo que abriga informações baseadas em um banco de dados de dois monitoramentos distintos: o MABEL, monitoramento da colmeia artificial, e o PTQA, monitoramento da qualidade do ar no IFSC — câmpus Chapecó.

Objetivos do Projeto

  • Promover o uso de tecnologias livres e de baixo custo no monitoramento ambiental.
  • Integrar o ensino, a pesquisa e a extensão no desenvolvimento de soluções sustentáveis.
  • Contribuir com dados que auxiliem pesquisadores e a comunidade na preservação ambiental.
  • Estimular o aprendizado prático de programação, eletrônica e análise de dados entre os estudantes.

Funcionamento Geral

Cada sistema utiliza sensores conectados a uma placa **Arduino**, responsável pela coleta dos dados ambientais.

As informações são enviadas a um **servidor web na nuvem**, onde são armazenadas e processadas em um **banco de dados MySQL**.

A interface web, desenvolvida com **HTML, CSS e JavaScript**, exibe dashboards com gráficos e indicadores atualizados em tempo real.

O backend, feito em **PHP**, permite comunicação entre o hardware e o banco de dados.

Importância Científica e Social

O monitoramento ambiental é essencial para compreender os impactos das ações humanas sobre os ecossistemas.

No caso do **MABEL**, os dados obtidos ajudam a preservar espécies de abelhas nativas, fundamentais para a polinização e manutenção da biodiversidade.

O **PTQA** fornece informações que podem orientar políticas locais de saúde e sustentabilidade, ampliando a conscientização sobre a qualidade do ar e seus efeitos na população.

Tecnologias Utilizadas

  • **Front-end:** HTML5, CSS3 e JavaScript.
  • **Back-end:** PHP.
  • **Banco de Dados:** MySQL.
  • **Ambiente de Desenvolvimento:** Visual Studio Code e XAMPP.
  • **Controle de Versão:** Git e GitHub.

Equipe e Desenvolvimento

O protótipo está sendo desenvolvido de forma incremental durante o módulo IV da disciplina de Programação.

As atualizações e novas funcionalidades serão implementadas conforme os requisitos forem sendo refinados.

O trabalho é colaborativo, com cada integrante responsável por uma parte do sistema — desde a codificação até o design e a documentação.
